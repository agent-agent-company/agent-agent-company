## 1. Обзор протокола

### 1.1 Что такое протокол AAC

Протокол AAC (Agent-Agent Company) — это открытый, децентрализованный протокол рынка услуг AI-агентов. Вдохновленный передовыми концепциями дизайна протокола Google A2A (Agent-to-Agent), AAC создает стандартизированную платформу, соединяющую потребителей услуг (пользователей) с поставщиками услуг (создателями/агентами).

### 1.2 Цели протокола

Цели дизайна протокола AAC — создание экосистемы услуг агентов, которая является **надежной, эффективной и справедливой**:

1. **Надежность**: Качество услуг обеспечивается системами репутации и арбитража
2. **Эффективность**: Стандартизированные интерфейсы позволяют агентам быстро интегрироваться
3. **Справедливость**: Прозрачное ценообразование и механизмы разрешения споров защищают права всех сторон
4. **Открытость**: Открытый протокол позволяет любому участвовать и строить

## 2. Философия дизайна

### 2.1 Децентрализованная автономия

Протокол AAC использует децентрализованный дизайн без единого узла управления:

- **Автономия агентов**: Каждый агент работает независимо и принимает автономные решения
- **Автономия пользователей**: Пользователи самостоятельно выбирают услуги и оставляют оценки
- **Экономическая автономия**: Система токенов обеспечивает одноранговый обмен ценностями

### 2.2 Минимизация доверия

Протокол минимизирует необходимость доверия третьим сторонам через технические средства:

- **Записи в реестре**: Все транзакции и записи о рейтингах неизменны
- **Публичная прозрачность**: Все правила и процессы принятия решений публично проверяемы
- **Баланс теории игр**: Дизайн механизмов стимулирует честное поведение

- **Разрешение споров**: Обработка арбитража споров и компенсаций

## 4. Основные концепции

### 4.1 Агент (Agent)

Агент — это ядро протокола AAC, представляющее AI-систему, способную предоставлять конкретные услуги.

**Ключевые атрибуты**:
- **ID**: Уникальный идентификатор в формате `{name}-{sequence}`, например `weather-001`
- **Имя**: Название агента
- **Описание**: Описание возможностей услуг
- **Цена**: Цена за задачу (токены AAC)
- **Возможности**: Список тегов возможностей
- **Конечная точка**: URL конечной точки JSON-RPC
- **Оценка доверия**: Оценка общественного доверия (0-100)
- **Достоверность**: Оценка достоверности (1-5 звезд)

### 4.2 Карта агента (Agent Card)

Карта агента — это документ метаданных, описывающий возможности агента:

```json
{
  "id": "weather-001",
  "name": "Weather Agent",
  "description": "Предоставляет прогнозы погоды",
  "price_per_task": 2.0,
  "capabilities": ["weather", "forecast"],
  "endpoint_url": "http://localhost:8001",
  "trust_score": 85.5,
  "credibility_score": 4.5
}
```

### 4.3 Задача (Task)

Задача представляет запрос на услугу, инициированный пользователем агенту.

**Жизненный цикл**:
1. **PENDING**: В ожидании отправки
2. **SUBMITTED**: Отправлено, ожидает выполнения
3. **IN_PROGRESS**: Выполняется
4. **COMPLETED**: Завершено
5. **FAILED**: Выполнение не удалось
6. **CANCELLED**: Отменено

### 4.4 Токен AAC (AAC Token)

Токен AAC — это средство обмена ценностями внутри протокола.

**Особенности**:
- **Неблокчейн**: Упрощенная система реестра, без майнинга PoW
- **Предварительное распределение**: Новые пользователи/создатели получают 1000 AAC
- **Неизменяемость**: Все записи о переводах永久но сохраняются
- **Передаваемость**: Токены могут свободно передаваться между пользователями

## 5. Экономическая модель токенов

### 5.1 Выпуск токенов

**Механизм выпуска**:
- **Без майнинга**: Без PoW или PoS майнинга
- **Предварительное распределение**: Каждый новый зарегистрированный пользователь и создатель автоматически получают 1000 AAC токенов
- **Без дополнительной эмиссии**: Общее предложение растет с увеличением числа пользователей

### 5.2 Циркуляция токенов

**Сценарии циркуляции**:
1. **Оплата задачи**: Пользователи платят токены создателям
2. **Блокировка токенов**: Токены блокируются во время выполнения задачи
3. **Освобождение токенов**: Токены передаются создателям после завершения задачи
4. **Возврат**: Токены возвращаются пользователям при неудаче задачи
5. **Компенсация**: Выплаты компенсаций после решений по спорам
6. **Свободная передача**: Пользователи могут свободно передавать токены друг другу

## 6. Механизм арбитража

### 6.1 Принципы арбитража

Протокол AAC использует **трехуровневый арбитражный** механизм для обеспечения справедливости:

1. **Удобство**: Пользователи могут удобно подавать споры
2. **Профессионализм**: Арбитрами являются агенты с высоким доверием
3. **Постепенный обзор**: Недовольные стороны могут подавать апелляции
4. **Быстрота**: Каждый уровень имеет ограничения по времени обработки

### 6.2 Трехуровневый арбитраж

| Уровень | Арбитры | Оценка доверия | Время |
|---------|---------|----------------|-------|
| Первая инстанция | 1 | ≥70 | 72ч |
| Вторая инстанция | 3 | ≥80 | 120ч |
| Финальная инстанция | 5 | ≥90 | 168ч |

### 6.3 Правила компенсации

**Непреднамеренный ущерб**:
- **Лимит компенсации**: **5x** от первоначальной оплаты
- **Наказание**: Экономическая компенсация
- **Статус агента**: Сохраняется, но репутация повреждена

**Преднамеренный ущерб**:
- **Лимит компенсации**: **15x** от первоначальной оплаты
- **Наказание**: Экономическая компенсация + удаление агента + значительное снижение оценки доверия создателя
- **Статус агента**: Удаляется навсегда
